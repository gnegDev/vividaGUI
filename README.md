# Вивида

Система оптимизации плана лечении глиобластомы.

Врач вводит данные пациента в клиентское приложение — оно отправляет их в локальный ML-сервис, который моделирует рост опухоли и возвращает рекомендацию по оптимизации протокола лечения. Результаты сохраняются в локальную базу данных.

---

## Компоненты

### `app/` — клиентское приложение

C# / .NET 10 / Avalonia UI. MVVM-архитектура, карточки пациентов, отображение результатов анализа.

→ [Подробное описание](app/README.md)

### `model/` — ML-сервис

Python / FastAPI. Stacking ensemble (XGBoost + GBM + RandomForest + ExtraTrees + MLP), симуляция роста опухоли по модели Гомперца, grid-search оптимизация дозировок.

→ [Подробное описание](model/README.md)

## Быстрый старт

**1. Скачать [архив](https://drive.google.com/file/d/1hvvvdLkFtKNuCw6LWm5ar7nnSlxJI8w7/view?usp=drive_link) и распаковать в `model/vivida/`**

**2. Запустить ML-сервис** (из папки `model/vivida/`):

```bash
docker build -t gbm-api .
docker run -p 8080:8080 gbm-api
```

**3. Запустить приложение** (из папки `app/vivida/`):

```bash
dotnet run
```

Приложение подключается к сервису на `http://localhost:8080`. Статус соединения — меню «Сервис → Статус сервиса».
